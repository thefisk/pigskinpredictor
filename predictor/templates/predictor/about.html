Pigskin Predictor was born out of curiosity and built with passion
In 2018, while working as a Windows infrastructure support guy (I used to do a lot of Perfmon & Wireshark analysis, memory dumps etc), I set out to expand my skillset by taking the edX course "Introduction to Computer Science and Programming Using Python" offered by MIT. Following on from this, I found out about Django and wondered what I could put together to take my Python skills further, and to start learning Django the best way; by making something.

I am a huge fan of the NFL and, in particular, the Chicago Bears. In the 2018/2019 season I played a weekly prediction game, jointly run by a friend's cousin. The game involved emailing in weekly predictions, which the administrators added to a spreadsheet. The game results were then manually added to the spreadsheet to produce the scores. But this was prone to the occasional human error and clearly took the administrators a long time to manage every week. So I thought, "there's got to be a better way!"

That was the lightbulb moment - why not try to use Python and Django to make a fully automated predictions game? One that automatically pulls in the schedule information from the NFL, pulls in the results week-by-week, updates the scores based on those results, and shows an accurate score table at all times?

That's what I've tried to build here and honestly, the process of putting this site together was the most fun I'd had with a computer in a long time.

As we head into the third season of Pigskin Predictor, I've added a raft of features and improvements including database performance tweaks, caching pages and data in Redis, using Chart.JS to add form graphs to profile pages, and adding asynchronous scripts to do jobs like email pick confirmations.

In 2021 a new 'Sunday Live' feature was added which polls the ESPN API every 60 seconds and provides a (more or less) live, moving scoretable for our predictions thanks to some VueJS magic. The backend only calls ESPN once every minute, while page requests here call on our own API which holds the current scores and is able to flag when a game has been updated. Building that was a lot of fun for a geek like me :)
